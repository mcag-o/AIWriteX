# AIWriteX 项目介绍与拆解建议

本文用于帮助你快速理解项目 [`AIWriteX`](README.md) 的整体结构、每个部分的组成与作用，并判断哪些模块适合保留、抽取、重写或直接舍弃，以便后续做你自己的二次开发。

---

## 1. 项目整体定位

从结构上看，[`AIWriteX`](README.md) 不是一个单纯的脚本仓库，而是一个“**桌面壳 + 本地 Web 服务 + 内容工作流 + 模板系统 + 平台发布**”的完整应用。

它的大致目标是：
- 获取选题或热搜
- 检索资料 / 借鉴参考文章
- 调用大模型生成文章
- 进行创意改写和风格变换
- 输出为 [`Markdown`](src/ai_write_x/core/unified_workflow.py:80) / [`HTML`](src/ai_write_x/adapters/platform_adapters.py:123)
- 保存到本地
- 选择性发布到公众号等平台

如果你要做自己的“自用流程”，真正值得关注的是它的**工作流骨架**，而不是完整 GUI 产品形态。

## 3. 启动链路：项目是怎么跑起来的

### 3.1 GUI 模式链路

1. 从 [`main.py`](main.py) 启动。
2. 调用 [`check_license_and_start()`](src/ai_write_x/license/__init__.py:7)。
3. 开源版本里，这个函数直接调用 [`gui_start()`](src/ai_write_x/license/__init__.py:18)。
4. GUI 真正由 [`WebViewGUI`](src/ai_write_x/web/webview_gui.py:22) 负责启动。
5. [`WebViewGUI.start()`](src/ai_write_x/web/webview_gui.py:154) 会：
   - 先启动本地 Uvicorn 服务
   - 再创建本地桌面窗口
   - 再把网页 UI 嵌进桌面壳

也就是说，这不是传统原生 GUI，而是：
- 后端：[`FastAPI`](src/ai_write_x/web/app.py:68)
- 前端：[`HTML/CSS/JS`](src/ai_write_x/web/static)
- 桌面壳：[`pywebview`](src/ai_write_x/web/webview_gui.py:4)

### 3.2 无 UI 模式链路

另一条主链路是 [`ai_write_x_main()`](src/ai_write_x/crew_main.py:195)。

它的作用是：
- 读取配置
- 校验 API / 平台参数
- 准备环境变量
- 调用 [`ai_write_x_run()`](src/ai_write_x/crew_main.py:139)
- 最后进入真正业务执行 [`run()`](src/ai_write_x/crew_main.py:115)

如果你要做自用工作流，这条链路比 GUI 链路更值得抽取。

## 4. 核心源码目录 [`src/ai_write_x`](src/ai_write_x)

这个目录是项目真正的业务核心。

### 4.1 [`src/ai_write_x/config`](src/ai_write_x/config)

#### 组成
- [`config.py`](src/ai_write_x/config/config.py)
- [`config.yaml`](src/ai_write_x/config/config.yaml)
- [`aiforge.toml`](src/ai_write_x/config/aiforge.toml)

#### 作用
这是项目的“配置中枢”。

重点类是 [`Config`](src/ai_write_x/config/config.py:34)，负责：
- 加载默认配置
- 合并用户配置
- 提供统一访问接口
- 校验配置合法性
- 保存配置到文件

你可以把它理解成项目的“全局运行态配置单例”。

#### 价值判断
**非常值得保留思路，但不一定值得整文件照搬。**

值得学习的原因：
- 大模型配置集中管理
- 发布平台配置集中管理
- 模板配置集中管理
- 写作参数集中管理

如果你自己二开，建议保留“统一配置中心”思想，但可以把 [`Config`](src/ai_write_x/config/config.py:34) 砍瘦，去掉你不用的平台和 UI 字段。

**这是最值得抽取的一级模块。**

如果你只想保留 20% 最有价值代码，那这一层至少值其中一半。

### 4.4 [`src/ai_write_x/creative`](src/ai_write_x/creative)

#### 组成

#### 作用
核心类是 [`DimensionalCreativeEngine`](src/ai_write_x/creative/dimensional_engine.py:19)。

这个模块负责：
- 定义创意维度
- 自动或手动选择维度组合
- 检查维度兼容性
- 生成创意提示词
- 在基础内容上做风格变换

- 做“去 AI 味”或“创意增强”

那么这一层很值得保留。

如果你只是做单一产出链路，这层可以后置，先不接。

## 8. 项目数据流：建议你如何理解整个系统

可以把它看成下面这条流水线：

1. 输入参数进入 [`ai_write_x_main()`](src/ai_write_x/crew_main.py:195)
2. 初始化系统 [`setup_aiwritex()`](src/ai_write_x/core/system_init.py:38)
3. 进入 [`UnifiedContentWorkflow.execute()`](src/ai_write_x/core/unified_workflow.py:133)
4. 用 [`AIForgeSearchTool`](src/ai_write_x/tools/custom_tool.py:133) 做搜索或参考文章抽取
5. 用 [`ContentGenerationEngine`](src/ai_write_x/core/content_generation.py:16) 生成基础稿
6. 用 [`DimensionalCreativeEngine`](src/ai_write_x/creative/dimensional_engine.py:19) 做创意改写
7. 用 [`PlatformAdapter`](src/ai_write_x/adapters/platform_adapters.py:74) 做平台格式化
8. 用模板系统生成或适配 HTML
9. 保存到 [`output`](output)
10. 可选发布到公众号等平台

这个数据流非常适合你拆成自己的“自用流程骨架”。

- 简单 FastAPI
